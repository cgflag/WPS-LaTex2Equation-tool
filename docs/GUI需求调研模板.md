#### 你问：装了 MathType 是不是就替代本工具了？

**简要结论：不能替代大多数你的工作。**

| 对比 | MathType | 本工具 |
|------|----------|--------|
| 场景 | 手动插入/编辑单个公式 | **整篇 docx 里已有大量 `$...$` 文本**（AI/ LaTeX 导出） |
| 输入 | 在 Word 里点公式编辑器 | **批量**把 `$...$` → 原生 OMML |
| 与 WPS | 偏 Word 生态，WPS 支持有限 | 输出 OOXML，**WPS 直接可编辑** |
| 费用 | 商业软件 | 开源 / 你可低价分发 |
| 块级居中编号 | 需自己排版 | **自动**制表位 + (1)(2)… |

MathType 适合「精修少数公式」；本工具适合「AI 生成说明书后一次性铺公式」。两者可并存，不冲突。

## 八、验收标准 【建议填写】

1. 无 Python 环境，双击 exe，拖入 `demo_before.docx`，得到 `_公式版.docx`，WPS 中公式可双击编辑。  
2. 本机无 XSL 时（或重命名 XSL 测试），启动即**阻塞**并显示中文说明，不能误点转换。  
3. 转换结束显示「成功 N / 失败 M」；M>0 时可看到失败公式原文列表。  
4. 默认输出为 `*_公式版.docx`，**不覆盖**原文件。  
5. 块级公式在 WPS 中**居中**，右侧有 (1)(2)…

**验收样例**： [x] 仅用 `examples/demo_before.docx`（【建议】）
